\paragraph{Previous Internal Solution}

In addition to these tools, the company had previously developed an internal system intended to centralize file storage. This system provided basic file management capabilities, such as file upload, organization, and editing through an integrated document viewer.

However, it remained limited in scope. It mainly addressed file storage and did not include event management, task assignment, advanced search, structured notifications, or a complete role-based access control mechanism. As a result, it did not fully support structured collaboration or controlled operational follow-up.

% -------------------------------------------------------
\subsection{Critique of the Existing System}

The existing environment presents several limitations that affect the efficiency and organization of football club operations.

\begin{itemize}
    \item \textbf{Fragmentation:} Staff members often use disconnected platforms, which can lead to scattered files and duplicated information.

    \item \textbf{Limited role-based access control:} Existing tools may provide basic sharing options, but they do not offer a unified permission model adapted to club roles.

    \item \textbf{Limited collaboration features:} The previous internal solution focused mainly on file storage and did not support event planning, task assignment, notifications, or collaborative workflows.

    \item \textbf{Limited search and organization:} Users may need to manually browse folders or separate tools to find relevant information.

    \item \textbf{Lack of unified traceability:} The tools used do not provide a centralized audit trail adapted to the operational needs of the platform.
\end{itemize}

Beyond these operational issues, the existing environment also lacks a centralized governance layer for identity, permissions, and data consistency. Each tool manages its own information separately, which may lead to duplication and inconsistencies. These limitations show the need for a more unified and controlled solution.

% -------------------------------------------------------
\subsection{Proposed Solution}

To address these limitations, this project proposes the development of \textbf{Football OS}, a unified platform designed to centralize and structure football club operations.

The proposed solution combines a foundational layer responsible for identity management, access control, canonical data, and traceability with an operational layer focused on daily collaboration workflows. Within this platform, the \textbf{Workspace module} provides features for managing documents, organizing events, consulting player-related information, receiving notifications, and searching workspace content.

\subsubsection*{Core Capabilities}

\begin{itemize}
    \item \textbf{Governance Foundation:} Management of identity, roles, permissions, canonical data, and audit mechanisms.

    \item \textbf{Workspace Services:} Management of documents, events, notifications, player profiles, and search.

    \item \textbf{Document Management:} Centralized access to files with structured organization.

    \item \textbf{Event Management:} Planning and coordination of club activities through a calendar-based workflow.

    \item \textbf{Controlled Access:} Role-based permissions applied to sensitive actions and resources.

    \item \textbf{Traceability:} Tracking important actions performed in the system.
\end{itemize}

\subsubsection*{Implementation Approach}

The system is designed as a modular platform that separates core management functions from user-facing features. This approach improves maintainability and supports future extensions while keeping the different parts of the platform integrated.
